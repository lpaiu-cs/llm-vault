# Software 1.0

인간 개발자가 명시적인 명령어 시퀀스(C, Python, Java 등)와 조건문 분기(if-else)를 활용하여 **원하는 도메인 규칙을 한 줄씩 수동으로 소스코드로 컴파일하는 전통적인 컴퓨팅 패러다임**. 연산의 주체가 사람이 설계한 결정론적 알고리즘과 로직 구조체이다.

## 핵심 메커니즘

1. 명시적 프로그래밍: 문제 해결 경로를 개발자가 직접 분석하여 예외 처리까지 완벽히 규정해야 함
2. 가독성: 소스코드가 인간 언어의 변형(추상 스택)이므로 논리 구조의 추적이 가능하며 수정이 직관적임
3. 확장성 병목: 복잡도가 일정 한계를 넘어서는 비전 인식, 자연어 이해 등의 영역에서는 예외 규칙의 무한 증식으로 인해 붕괴함

## 핵심 엣지

- `[[Software 2.0]] contradicts [[Software 1.0]]` — 가중치 공간 자동 탐색 패러다임과 인간 중심의 명시적 명령형 코드 체계는 상호 배타적인 주체 철학을 지님
